So I've been building a browser Tetris game called Glowtris for a while now, and I figured it's time to start writing about it.

Not because I think people desperately want to read my dev blog, but because the game has reached a point where interesting decisions are being made — and commit messages like `fix: daily board reset timing` don't really capture *why* things ended up the way they did.

## What this blog is about

**DEV** — Technical write-ups. Architecture decisions, bugs that took way too long to find, performance stuff. I'll try to explain why I did things a certain way, not just what I did.

**UPDATE** — Feature releases. Less formal than a changelog, more honest about what actually shipped vs. what I planned.

**GUIDE** — Tips for actually getting good at the game. T-Spins, combos, sprint strategies. That kind of thing.

**NOTICE** — This. General announcements.

---

Quick context if you stumbled here: Glowtris is a free, ad-free browser Tetris clone I've been building solo. It has daily challenges, sprint mode, leaderboards, and a push notification system that... mostly works.

It's publicly available at [glowtris.com](https://glowtris.com) but I still call it "beta" because the game engine doesn't yet meet the standard I want for a v1.0 release. T-Spin detection, proper rotation systems, frame-perfect input handling — there's still work to do.

I'm writing this blog partly to document that journey, and partly because writing forces me to think more clearly about what I'm building and why.

Hope some of it is useful or at least mildly interesting.
