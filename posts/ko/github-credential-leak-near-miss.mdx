군대 연등시간에 깃허브에 보안 키를 퍼블릭으로 올렸습니다.

경고 메일은 단 한 통도 오지 않았습니다.

그래서 저는 아무것도 모른 채 당당하게 Vercel 배포까지 신나게 눌렀습니다.

## 😱 사건의 전말

보통 깃허브에 키가 유출되면 GitGuardian이나 AWS 같은 곳에서 "너 지금 보안 키 노출됐어!"라고 경고 메일이 날아온다던데... **저는 경고 메일 같은 건 하나도 받지 못했습니다.** 

제가 퍼블릭으로 올린 건 다름 아닌 **보안 자격 증명(Credentials)**과 **Moshi 웹훅 토큰**이었습니다. 메일도 안 왔으니 저는 유출된 줄도 모르고 평온하게 개발을 이어갔죠.

## 🔥 왜 이게 무서운가? (최근 티빙 보안사고 예시)

불과 얼마 전, 2026년 6월 초에 발생한 **티빙(TVING) 1,300만 명 개인정보 유출 사태** 기억하시나요? 

그 대형 사고의 시작점도 해커의 고도화된 침투 기술이 아니라, **오픈소스 공유 플랫폼(GitHub)에 노출된 AWS 액세스 키**였습니다. 개발 과정에서 실수로 올라간 키를 해커가 탈취했고, 그걸로 내부 데이터베이스에 접근해서 엄청난 쿼리를 돌려 1,300만 명의 개인정보를 빼간 거죠. 결국 이 실수 하나로 대규모 집단 소송까지 진행 중입니다.

제 퍼블릭 레포지토리에 올라간 토큰과 키를 누군가 봇으로 긁어갔다면? 제 계정으로 암호화폐 채굴기를 수백 대 돌려서 하룻밤 새 수천만 원의 요금 폭탄을 맞거나, 제 조그만 서버가 다 털렸을 수도 있습니다.

## 🤔 왜 경고 메일이 안 왔을까?

지금도 생각하면 아찔하지만, 왜 시스템이 감지하지 못했을까요? 제 추정은 이렇습니다:

- **Moshi 웹훅 토큰은 표준 형식이 아닙니다.** GitGuardian이나 GitHub의 Secret Scanning은 `AKIA`로 시작하는 AWS 키, GitHub PAT 등 **패턴이 이미 알려진** 키들을 우선 감지합니다. 커스텀 서비스의 토큰은 패턴 인식이 안 되어 스캐너를 무사 통과할 확률이 높습니다.
- **당시 제 레포에 보안 스캐너가 연동되어 있지 않았습니다.**

결론: **"경고가 안 오면 안전하다"가 아닙니다. 경고 시스템 자체가 완벽하지 않습니다.**

## 🏃‍♂️ 나의 조치 내역 (소름 주의)

처음에는 유출된 사실도 모른 채, 아래 커맨드로 당당하게 코드를 올리고 배포했습니다.
```bash
gh repo create tetris --public --push
vercel deploy
```
이 사실이 가장 소름 돋는 부분입니다.

하지만 뒤늦게 `robot.md`와 `CLAUDE.md` 파일 등에 **Moshi 웹훅 토큰(`9W8b67...`)이 하드코딩되어 노출된 것**을 깨닫고, 벼락 맞은 듯이 다음 조치들을 실행했습니다. 저처럼 키 노출을 겪으신 분들은 아래 순서대로 빠르게 대처하시길 바랍니다.

### 1. 로컬 백업 및 격리
노출된 토큰과 Admin 비밀번호 등 모든 민감한 정보를 로컬의 `CREDENTIALS.md` 파일에 서버 전용으로 백업했습니다.

### 2. `.gitignore` 추가
이 백업 파일이 또 깃허브에 올라가는 대참사를 막기 위해 `.gitignore`에 즉시 추가했습니다.
```text
# .gitignore
.env
.env.local
.env.*

# 민감한 백업 문서
CREDENTIALS.md
```

### 3. 토큰 즉시 폐기 및 재발급
노출된 Moshi 토큰을 즉시 무효화하고 새로운 토큰(`G5aOFC...`)을 발급받았습니다. 이미 깃허브 커밋 히스토리에 남은 이상 파일을 지우는 것만으로는 안전하지 않습니다. **무조건 토큰 자체를 폐기해야 합니다.**

### 4. 새 토큰 연동 및 테스트
`curl`로 새 웹훅 토큰이 정상 작동하는지 테스트한 후, 시스템에 안전하게 재연동했습니다. 
```bash
moshi-hook pair --token G5aOFC...
```

## 💡 교훈 및 실전 체크리스트

"나는 안 그러겠지" 하다가 진짜 한 방에 훅 갈 수 있습니다. 지금 당장 따라 할 수 있는 2가지를 추천합니다.

### 1. 글로벌 .gitignore 설정 (한 번만 해두면 평생 적용)
```bash
echo -e ".env\n.env.local\n.env.*.local\nCREDENTIALS.md" >> ~/.gitignore_global
git config --global core.excludesfile ~/.gitignore_global
```

### 2. GitGuardian 무료 연동 (5분 컷)
소 잃고 외양간 고치는 격이지만, 이 사태 이후 즉시 GitHub 레포지토리에 [GitGuardian](https://www.gitguardian.com/)을 연동하여 향후 키 노출을 봇이 자동으로 감지하고 차단할 수 있도록 세팅했습니다.

오늘도 깃허브에 코드를 푸시하기 전에 `git status`를 세 번 확인하는 습관을 가져야겠습니다.
