> **요약:** AI를 활용한 UI 개발 시, 기본적인 레이아웃과 톤앤매너는 프롬프트로 빠르게 뽑아낼 수 있습니다. 하지만 디테일한 간격(Spacing), 폰트 스케일, 다크모드 색상 반전, 그리고 마이크로인터랙션은 여전히 디자이너의 예리한 시각적 보정과 끝없는 피드백을 요구합니다.

7년 차 디자이너입니다. 서울과 뉴욕에서 구르며 여백과 타이포그래피는 빠삭하게 익혔습니다.

하지만 코딩은 완전 젬병입니다. `npm install`이 왜 터지는지 지금도 모릅니다. 

그런 제가, 코딩 한 줄 없이 오직 클로드(Claude)에게 **말로만 입을 털어서** 브라우저 테트리스 게임 **[글로우트리스](www.glowtris.com)**와 이 블로그를 통째로 만들었습니다.

바이브코딩(Vibe Coding)이 마법처럼 다 해주는 건 절대 아닙니다. 하지만 제 작업 방식은 이렇게 바뀌었습니다.

1. **머릿속 그림 설명하기** → 2. **AI가 뚝딱 만들기** → 3. **보고 "이거 아닌데?" 피드백 주기** → 4. **수정하기**

노동은 AI가 하지만, **'이게 맞고 저건 틀리다'라는 판단은 사람의 몫**입니다. 특히 UI/UX는 7년 동안 현업에서 구르며 눈에 박힌 디자인 감각이 절대적으로 필요했죠. 코딩을 몰라서 입도 뻥긋 못하던 시절과는 차원이 다릅니다. 미친 듯이 손이 빠른 시니어 개발자를 노예... 아니, 파트너로 둔 기분입니다.

## 글로우트리스 색상 시스템: 찰떡같이 알아듣는 법

글로우트리스의 핵심 컨셉은 딱 하나였습니다. **"네온 아케이드 같으면서 20분 넘게 봐도 눈이 안 아플 것."**

저의 첫 프롬프트는 이랬습니다.
> "배경을 매우 어두운 네이비, 거의 블랙으로 만들어줘. 블럭 네온 글로우 색상은 시안, 마젠타, 노랑, 초록. UI는 SF 영화 어두운 인터페이스처럼 — 깔끔하고, 미니멀하고, 빛나는."

클로드가 배경에 `#000000`을 깔고 네온 색상을 입혀줬습니다. 근데 문제가 생겼습니다. **순수 블랙은 글로우 효과를 죽여버립니다.** 빛은 완전 블랙이 아니라 어두운 블루-블랙 위에서 진가를 발휘하거든요. 즉시 태클을 걸었습니다.

> "배경이 너무 밋밋해. 딥하고 채도 낮은 네이비여야 해 — `#04041e` 같은 거. 테트리스 블럭은 진짜 빛을 내뿜는 것처럼 보여야 한다고."

결과는 대성공이었습니다. 제가 즉흥적으로 뱉은 `#04041e`는 실제 프로덕션 값이 되었습니다.

## 타이포그래피 시스템: 교수님이 옳았습니다

로고 폰트는 묵직하고 기하학적인 SF 서체 `Orbitron`을 썼습니다. 백 미터 밖에서도 "우주 게임" 냄새가 나죠. 

근데 이걸 본문 크기로 쓰면 가독성이 개판이 됩니다. 그래서 폰트 시스템을 완전히 분리해 달라고 요구했습니다.

> "헤더랑 로고는 `Orbitron`. 본문이랑 UI는 한국어 `Pretendard`, 영어 `system-ui`. 코드 같은 모노스페이스 요소는 `JetBrains Mono`."

디스플레이 / 본문 / 모노로 나누는 건 대학교 UI 수업 때 나재휘 교수님께서 귀에 못이 박히도록 강조하신 표준 패턴입니다. 선배들이 닦아둔 길을 AI도 찰떡같이 이해하고 적용하더군요.

## 프롬프트로 절대 안 되는 4가지

AI가 만능일까요? 아닙니다. 저는 눈알이 빠지도록 모니터를 노려보며 이 4가지는 항상 **직접** 수정했습니다.

1. **간격 (Spacing)**: AI가 뱉은 간격은 항상 너무 좁거나 멍청하게 넓습니다. 로고 자간은 줄이고, 섹션 여백은 늘리고, 카드 패딩은 시각적 보정을 거쳐야 합니다.
2. **타이포그래피 스케일**: AI는 수학적으로 완벽한 스케일을 주지만, 눈으로 보면 밋밋합니다. 헤딩 크기는 억지로 더 키우고 본문은 줄여서 대비를 극대화해야 합니다.
3. **색상 상호작용**: AI는 단일 색상은 잘 뽑지만, 여러 색상이 겹칠 때(다크모드 활성 버튼 반전 등) 디테일을 밥 먹듯이 놓칩니다.
4. **마이크로인터랙션**: AI의 호버 애니메이션은 너무 소심하거나 너무 오두방정을 떱니다. 저는 무조건 `0.15s ease`에 빠른 반응, 자신감 있는 호버를 밀어붙였습니다.
